HTML Structure

Documentation about the HTML5 standard can be found here. Especial the index at the bottom is usable because it contains all the tags.

The p tag stands for paragraph.

HTML5 document type declaration tag !doctype html. All document start with an n tag, a good thing could be to have a language attribute lang="en" so the web browser knows the language the page is using. Then the document needs a head and body tag. In the head the title tag and meta tag with the attribute charset="utf-8".

For checking the outline of your HTML5 document, one can user HTML5 outliner or HeadingsMap for chrome.

The figure element is used for illustrating things, but it doesn't have to be an image.

The div element should only be a user when you need to group other elements for style purposes.

The address element represents the nearest article or body element.

The time element should be used for time and date information. If it's not in a valid format you need to add the valid date to a datetime attribute. Time should be in the current timezone the author is in.

Links is set with an a or link tag and can have a rel attribute to tell how the search engine, etc. should follow the link, like:

  • next: next document in a series
  • prev: last document in a series
  • nofollow: tell the search engine that this document is not relative to the given link.
  • stylesheet: for a link for importing a stylesheet.

The meta element set the character set with the charset attribute, but it can also be set to a name attribute to give meta information about the site, description is usually used to give as summary as the search result. The name attribute can also be used to format how the page should look if someone twittered or link it on facebook.

The class attribute can be used at several places and it used for example CSS to identify the tag type. The id attributes should only be used once per document to identify a specified element.

Landmark roles, a role for a section could be application, banner, complementary,contentinfo, form, main, navigation and search. These are used by screen reader.

Schema meta tag is used by the search engine for categorising a section. An example could be that a movie can have a name, director, genre and trailer. These attributes are defined at schema.org.

After adding the metadata tag, the presentation of the search result of the page can be tested by using the Structured Data Testing Tool in google's Webmaster.

New elements tag for HTML5 for a section, except for the old h1-6 and div, is article, aside, nav and section. There is a new tag for grouping element: footer, header and main. Here is an example how to use the new tags element:

<head>
  <meta charset="utf-8"/>
  <meta name="description" content="This page describes information about trees."/>
</head>
<body>
  <header>
    <h1>Exploring Trees</h1>
    <nav>
      <!-- page navigation -->
    </nav>
  </header>
  <main>
    <article id="types">
      <h2>Tree types</h2>
      <section>
        <h3>Deciduous tree</h3>
        <h4>White Spruce</h4>
        <h4>Balsam Fir</h4>
      </section>
    </article>
    <article id="care">
      <h2>Taking Care of Trees</h2>
      <section>
        <h3>Basic tree mainatenace</h3>
      </section>
      <section>
        <h3>Proper water levels</h3>
      </section>
    </article>
    <aside>
      <h2>Learning about trees</h2>
    </aside>
  </main>
  <footer>
    <h2>Contact me</h2>
    <p>Here!</p>
  </footer>
</body>
